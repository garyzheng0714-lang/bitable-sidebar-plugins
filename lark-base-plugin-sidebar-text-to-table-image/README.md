# 文字转表格图片 (Text to Table Image) - 飞书多维表格插件

这是一个飞书多维表格（Lark Bitable）的侧边栏插件，旨在将文本字段中的结构化数据（如营养成分表、Markdown 表格）自动渲染为美观的表格图片，并上传至云存储（阿里云 OSS 或火山引擎 TOS），最后将图片链接回填至表格中。

## ✨ 核心功能

*   **文本转图片**：支持将 Markdown 格式或特定格式的文本数据解析并渲染为表格图片。
*   **智能解析**：
    *   支持标准 Markdown 表格语法。
    *   支持自定义营养成分表格式（`# header:` 开头，`|` 分隔）。
    *   自动处理单位验证和 NRV% 计算。
*   **多云存储支持**：
    *   支持 **阿里云 OSS** (Aliyun Object Storage Service)。
    *   支持 **火山引擎 TOS** (Volcengine TOS)。
*   **批量处理**：
    *   支持一键处理当前视图内的所有可见记录。
    *   **并发控制**：内置并发限制（默认为 3），防止浏览器卡顿。
    *   **任务管理**：支持中途停止批量任务。
*   **UI/UX 优化**：
    *   符合飞书边栏插件设计规范。
    *   支持深色/浅色主题自适应。
    *   提供详细的进度反馈和错误提示。
    *   **安全配置**：API 密钥等配置存储在本地浏览器（LocalStorage），支持密码隐藏显示。

## 🛠️ 安装与开发

### 1. 环境准备

*   Node.js (推荐 v14 或 v16)
*   npm 或 yarn

### 2. 安装依赖

```bash
npm install
```

### 3. 启动开发服务器

```bash
npm run dev
```

启动后，控制台会显示本地服务器地址（通常是 `http://localhost:5173`）。

### 4. 在飞书多维表格中调试

1.  打开飞书多维表格。
2.  点击右上角的 **扩展脚本** -> **添加自定义插件**。
3.  选择 **侧边栏插件**。
4.  填入本地开发服务器地址（例如 `http://localhost:5173/`）。
5.  保存并运行。

## ⚙️ 配置说明

首次使用时，需要在插件设置面板中配置云存储服务的访问密钥：

1.  **服务商**：选择 Aliyun OSS 或 Volcengine TOS。
2.  **AccessKeyId**：您的云服务访问密钥 ID。
3.  **AccessKeySecret**：您的云服务访问密钥 Secret（加密存储）。
4.  **Bucket**：存储桶名称。
5.  **Region**：区域代码（例如 `oss-cn-beijing` 或 `cn-beijing`）。
6.  **Endpoint**（可选）：自定义节点地址。

点击 **保存配置** 后，配置信息将安全存储在您的本地浏览器中。

## 📖 使用指南

1.  **准备数据**：
    *   在多维表格中准备一个 **文本字段** 作为源数据。
    *   准备一个 **文本字段** 作为目标字段（用于存储生成的图片 URL）。
2.  **选择字段**：
    *   在插件界面选择对应的“源文本字段”和“目标链接字段”。
3.  **开始转换**：
    *   **单条处理**：在表格中选中一行，点击“生成并上传”。
    *   **批量处理**：点击“批量”按钮，插件将自动处理当前视图下的所有记录。
4.  **查看结果**：
    *   处理完成后，目标字段将填入图片的 URL 地址。若字段类型设置为“文本”，飞书会自动识别链接；若为“附件”字段，请确保逻辑适配（当前版本主要针对文本链接回填）。

## 📝 数据格式示例

### 营养成分表格式

```text
# header: 项目 | 每份（15毫升） | 营养素参考值%
能量 | 87千焦 | 0.01
蛋白质 | 1.2克 | 0.02
脂肪 | 0克 | 0.0
```

### Markdown 表格格式

```markdown
| 产品参数 | 内容 |
|---|---|
| 品牌 | 养命酒制造 |
| 风味 | 蜂蜜 |
```

构建生产环境代码：

```bash
npm run build
```

构建产物将输出到 `dist` 目录。
